# <a name="licensing-and-branches-for-system-center-configuration-manager"></a>Licencjonowanie i gałęzie dla programu System Center Configuration Manager

*Dotyczy: System Center Configuration Manager (bieżącej gałęzi) (długoterminowej obsługi oddziału)*

Aby dowiedzieć się więcej o wymaganiach dotyczących licencjonowania opcje instalacji dostępne w wersji 2016 października programu System Center Configuration Manager wersji 1606, należy użyć w tym temacie. Te opcje instalacji obejmują wersji bieżącej gałęzi 1606, długoterminowe obsługi gałęzi (LTSB) i instalację w wersji ewaluacyjnej do wersji bieżącej gałęzi 1606.

**Omówienie licencjonowania:**   
Klienci z active Software Assurance (SA) dla licencji programu System Center Configuration Manager lub prawa równoważne subskrypcji, począwszy od 1 października 2016 mają prawa do użytkowania wydania wersji 1606 2016 października programu System Center Configuration Manager. Użytkownicy z prawami do System Center Configuration Manager w lub po 1 października 2016 będą dostępne dwie opcje licencjonowanego podczas instalacji: Bieżącej gałęzi i długoterminowej obsługi gałęzi (LTSB).


**Charakterystyka licencjonowania:**  
[Zakończenie warunków użytkowania produktów zakupu za pośrednictwem programów licencjonowania zbiorowego firmy Microsoft można znaleźć tutaj](http://go.microsoft.com/fwlink/?LinkId=800052).

Zainstaluj Technical Preview przy użyciu innego nośnika niż licencjonowanej wersji. Aby uzyskać więcej informacji, zobacz dokumentację [Technical Preview](/sccm/core/get-started/technical-preview).


## <a name="licensed-branches"></a>Licencjonowanego gałęzi
Klienci z active Software Assurance (SA) dla licencji programu System Center Configuration Manager lub prawa równoważne subskrypcji, począwszy od 1 października 2016 mają prawa do użytkowania wydania wersji 1606 2016 października programu System Center Configuration Manager. Klienci z prawami do System Center Configuration Manager w wersji 1606 w lub po 1 października 2016 będą dostępne dwie opcje licencjonowanego podczas instalacji:
-    **Bieżącej gałęzi**
-    **Długoterminowe obsługi gałęzi (LTSB)**


Zobacz tabelę w poprzedniej sekcji, aby uzyskać więcej informacji.


## <a name="software-assurance-agreements-and-system-center-configuration-manager"></a>Software Assurance umowy i System Center Configuration Manager
Stan Software Assurance w ramach licencji programu System Center Configuration Manager lub prawa równoważne subskrypcji, w lub po 1 października 2016 określa gałąź można zainstalować i używać.


### <a name="software-assurance-and-the-current-branch"></a>Software Assurance i bieżącej gałęzi
Prawa do używania bieżącej gałęzi System Center Configuration Manager może być udostępniane przez:
-  **System Center:** Klienci z active SA na System Center Standard lub Datacenter licencji można zainstalować i używać opcji bieżącej gałęzi programu System Center Configuration Manager.

-  **System Center Configuration Manager:** Klientów z active SA dla licencji programu System Center Configuration Manager, lub prawa równoważne subskrypcji można zainstalować i używać opcji bieżącej gałęzi programu System Center Configuration Manager.

Jeśli masz aktywne SA w licencji programu System Center Configuration Manager (lub równoważne subskrypcji prawa) w lub po 1 października 2016:
- Można zainstalować i używać bieżącej gałęzi.
- Jeśli zezwolisz SA lub subskrypcja wygasną, należy odinstalować bieżącej gałęzi.

### <a name="software-assurance-and-the-ltsb"></a>Software Assurance i LTSB
 Jeśli masz aktywne SA w licencji programu System Center Configuration Manager (lub równoważne subskrypcji prawa) w lub po 1 października 2016:
 - Można zainstalować i używać LTSB. Klientów mających prawa wiecznej programu System Center Configuration Manager lub który umożliwia ich SA lub subskrypcji wygaśnięcie, można zainstalować wersję programu System Center Configuration Manager LTSB są aktualne w chwili wygaśnięcia.

LTSB opiera się na bieżącą wersję 1606 i ma następujące ograniczenia:
  - Nie jest obsługiwane do konwersji bieżącej gałęzi na LTSB. Jeśli masz obecnie gałęzi bieżącej lokacji, należy zainstalować LTSB jako nowej lokacji.  

  - LTSB nie obsługuje wszystkie funkcje w bieżącej gałęzi. Ograniczenia są wyszczególnione w [wprowadzenie do gałęzi obsługi długoterminowe](introduction-to-the-ltsb.md) i jego dokumentacji powiązanych. Te ograniczenia obejmują funkcję ograniczonego zestawu, ograniczone opcje uaktualnienia i zasady świadczenia pomocy technicznej oddzielny produkt.  


### <a name="software-assurance-expiration-date"></a>Data wygaśnięcia Software Assurance
Począwszy od wersji 2016 października nośników linii bazowej 1606 wersji programu System Center Configuration Manager, można określić datę wygaśnięcia umowy Software Assurance. **Data wygaśnięcia Software Assurance** jest opcjonalna wartość, można określić jako wygodny monitu podczas uruchamiania Instalatora programu Configuration Manager lub nowszy z w ramach programu Configuration Manager konsoli.

>  [!NOTE]   
>  Firma Microsoft nie obsługuje sprawdzania daty wygaśnięcia i nie używa tej daty w celu weryfikacji licencji.  W takim przypadku go użyć jako przypomnienie od daty wygaśnięcia. Jest to przydatne, ponieważ program Configuration Manager okresowo sprawdza, czy nowe aktualizacje oprogramowania oferowanych online, a stan licencji programu Software Assurance powinna być bieżącego Aby kwalifikować się do korzystania z tych dodatkowych aktualizacji.    

**Aby określić datę:**
- Po uruchomieniu Instalatora z nośnika programu System Center Configuration Manager wersji 1606 linii bazowej, można określić wartość na **klucz produktu** w Kreatorze instalacji.

- W konsoli programu Configuration Manager w **właściwości ustawienia hierarchii**, można określić wartość na **licencjonowania** kartę.

Aby uzyskać więcej informacji na temat licencjonowania Software Assurance i bieżącej gałęzi programu System Center Configuration Manager, zobacz [licencjonowania i gałęzie dla programu System Center Configuration Manager](/sccm/core/understand/learn-more-editions).
