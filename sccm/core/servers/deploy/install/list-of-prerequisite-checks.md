# <a name="list-of-prerequisite-checks-for-system-center-configuration-manager"></a>Lista testów wymagań wstępnych programu System Center Configuration Manager

*Dotyczy: Program System Center Configuration Manager (Current Branch)*

W poniższych sekcjach opisano dostępne funkcje sprawdzania wymagań wstępnych.

Aby uzyskać informacje o używaniu narzędzia sprawdzania wymagań wstępnych, zobacz [narzędzie sprawdzania wymagań wstępnych](prerequisite-checker.md).  

##  <a name="BKMK_Security"></a>Sprawdzanie wymagań wstępnych dotyczące praw zabezpieczeń  
W poniższej tabeli wymieniono kontroli wykonywanych przez narzędzie sprawdzania wymagań wstępnych w przypadku praw zabezpieczeń.

|Operacji sprawdzania|Wyjaśnienie|Ważność|Stosowanie lokacji|
|---|---|---|---|
|**Prawa administratora w centralnej lokacji administracyjnej**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **administratora** uprawnienia na komputerze centralnej lokacji administracyjnej. |Błąd|Lokacja główna|
|**Prawa administracyjne w rozszerzanej lokacji głównej**|Sprawdza, czy konto użytkownika uruchamiającego instalację ma **administratora** prawa w autonomicznej lokacji głównej, która zostanie rozszerzona.|Błąd|Centralna lokacja administracyjna|
|**Prawa administracyjne w systemie lokacji**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **administratora** uprawnienia na komputerze serwera lokacji. |Błąd|Centralna lokacja administracyjna, <br>Lokacja główna <br>Lokacja dodatkowa|
|**Prawa administracyjne na komputerze CAS rozszerzanej lokacji głównej**|Sprawdza, czy konto komputera centralnej lokacji administracyjnej ma **administratora** prawa w autonomicznej lokacji głównej, która zostanie rozszerzona.|Błąd|Centralna lokacja administracyjna|
|**Połączenie z programem SQL Server w centralnej lokacji administracyjnej**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager w lokacji głównej do dołączenia do istniejącej hierarchii ma **sysadmin** rolę w wystąpieniu programu SQL Server dla centralnej lokacji administracyjnej.|Błąd|Lokacja główna|
|**Prawa administracyjne konta komputera serwera lokacji**|Sprawdza, czy konto komputera serwera lokacji ma **administratora** komputery punktu prawa w programie SQL server i zarządzania.|Błąd|Lokacja główna <br>Serwer SQL|
|**System lokacji do komunikacji z serwerem SQL**| Sprawdza, czy prawidłowa usługi nazwa główna (SPN) jest zarejestrowana w usługach domenowych Active Directory dla konta skonfigurowanego do uruchamiania usługi programu SQL Server dla wystąpienia programu SQL Server, który jest hostem programu Configuration Manager bazy danych lokacji. Aby zapewnić obsługę uwierzytelniania Kerberos, w Usługach domenowych Active Directory należy zarejestrować prawidłową nazwę SPN.|Ostrzeżenie|Lokacji dodatkowej <br>Punkt zarządzania|
|**Tryb zabezpieczeń programu SQL Server**|Sprawdza, czy program SQL Server jest skonfigurowany dla zabezpieczeń uwierzytelniania systemu Windows.|Ostrzeżenie|Serwer SQL|
|**Uprawnienia sysadmin serwera SQL**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **sysadmin** rolę w wystąpieniu programu SQL Server wybranym do zainstalowania bazy danych lokacji. To sprawdzenie nie powiedzie się także wtedy, gdy Instalator nie będzie mógł uzyskać dostępu do wystąpienia dla programu SQL Server w celu sprawdzenia uprawnień.|Błąd|Serwer SQL|
|**Uprawnienia sysadmin serwera SQL dla lokacji odniesienia**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **sysadmin** wystąpieniu roli programu SQL Server wybranym jako baza danych lokacji odniesienia. SQL Server **sysadmin** wymagane są uprawnienia roli do modyfikowania bazy danych lokacji.|Błąd|Serwer SQL|
