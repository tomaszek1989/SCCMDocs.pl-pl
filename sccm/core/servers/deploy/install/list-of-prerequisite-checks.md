# <a name="list-of-prerequisite-checks-for-system-center-configuration-manager"></a>Lista Sprawdzanie wymagań wstępnych dla programu System Center Configuration Manager

*Dotyczy: System Center Configuration Manager (bieżącej gałęzi)*

W poniższych sekcjach opisano dostępne funkcje sprawdzania wymagań wstępnych.

Aby uzyskać informacje o używaniu narzędzia sprawdzania wymagań wstępnych, zobacz [narzędzie sprawdzania wymagań wstępnych](prerequisite-checker.md).  

##  <a name="BKMK_Security"></a>Sprawdzanie wymagań wstępnych dotyczące praw zabezpieczeń  
W poniższej tabeli wymieniono kontroli wykonywanych przez narzędzie sprawdzania wymagań wstępnych dotyczące praw zabezpieczeń.

|Sprawdzania|Wyjaśnienie|Ważność|Stosowanie lokacji|
|---|---|---|---|
|**Prawa administratora w witrynie Administracja centralna**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **administratora** uprawnienia na komputerze witryny Administracja centralna. |Błąd|Lokacja główna|
|**Prawa administracyjne w rozszerzonej lokacji głównej**|Sprawdza, czy konto użytkownika uruchamiającego instalację ma **administratora** prawa w autonomicznej lokacji głównej, która zostanie rozszerzona.|Błąd|Centralna lokacja administracyjna|
|**Prawa administracyjne w systemie lokacji**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **administratora** uprawnienia na komputerze serwera lokacji. |Błąd|Witryny Administracja centralna <br>Lokacja główna <br>Lokacja dodatkowa|
|**Prawa administracyjne na komputerze CAS rozwiń węzeł lokacji podstawowej**|Sprawdza, czy konto komputera centralnej lokacji administracyjnej ma **administratora** prawa w autonomicznej lokacji głównej, która zostanie rozszerzona.|Błąd|Centralna lokacja administracyjna|
|**Połączenie z programem SQL Server w witrynie Administracja centralna**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager w lokacji podstawowej do dołączenia do istniejącej hierarchii ma **sysadmin** w wystąpieniu programu SQL Server dla witryny administracji centralnej.|Błąd|Lokacja główna|
|**Prawa administracyjne konta komputera serwera lokacji**|Sprawdza, czy konto komputera serwera lokacji ma **administratora** komputery punktu uprawnień na serwerze SQL i zarządzania.|Błąd|Lokacja główna <br>Serwer SQL|
|**System lokacji do komunikacji z serwerem SQL**| Sprawdza, czy prawidłowa usługi nazwa główna (SPN) jest zarejestrowana w usługach domenowych Active Directory dla konta skonfigurowanego do uruchamiania usługi programu SQL Server dla wystąpienia programu SQL Server obsługującym Menedżera konfiguracji bazy danych lokacji. Aby zapewnić obsługę uwierzytelniania Kerberos, w Usługach domenowych Active Directory należy zarejestrować prawidłową nazwę SPN.|Ostrzeżenie|Lokacja dodatkowa <br>Punkt zarządzania|
|**Tryb zabezpieczeń programu SQL Server**|Sprawdza, czy program SQL Server jest skonfigurowany dla zabezpieczeń uwierzytelniania systemu Windows.|Ostrzeżenie|Serwer SQL|
|**Prawa administratora systemu SQL Server**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **sysadmin** rolę w wystąpieniu programu SQL Server wybranym dla instalacji bazy danych lokacji. To sprawdzenie nie powiedzie się także wtedy, gdy Instalator nie będzie mógł uzyskać dostępu do wystąpienia dla programu SQL Server w celu sprawdzenia uprawnień.|Błąd|Serwer SQL|
|**Uprawnienia sysadmin serwera SQL dla lokacji odniesienia**|Sprawdza, czy konto użytkownika uruchamiającego instalację programu Configuration Manager ma **sysadmin** wystąpieniu roli programu SQL Server wybranym jako baza danych lokacji odniesienia. SQL Server **sysadmin** wymagane są uprawnienia roli do modyfikowania bazy danych lokacji.|Błąd|Serwer SQL|

|**Prawa administracyjne w punkcie zarządzania**|Sprawdza, czy konto komputera serwera lokacji ma **administratora** komputerze punktu prawa na punkt zarządzania i dystrybucji.|Ostrzeżenie|Punkt zarządzania|
|**Udział administracyjny (system lokacji)**|Sprawdza, czy wymagane udziały administracyjne są dostępne na komputerze systemu lokacji.|Ostrzeżenie|Punkt zarządzania|

|**Usługa BITS zainstalowana**|Sprawdza, czy usługa BITS jest zainstalowana w usługach IIS.|Ostrzeżenie|Punkt zarządzania|
|**Sortowanie bez uwzględniania wielkości liter na serwerze SQL**|Sprawdza, czy instalacja programu SQL Server używa sortowania bez uwzględniania wielkości liter, na przykład SQL_Latin1_General_CP1_CI_AS.|Błąd|Serwer SQL|
|**Sprawdź wersję i kod lokacji istniejącej autonomicznej lokacji głównej**|Sprawdza, czy witryna głównej planowana do rozszerzenia jest autonomiczną lokacją główną i ma tę samą wersję programu Configuration Manager, ale kod innej lokacji niż centralna lokacja administracyjna do zainstalowania.|Błąd|Witryny Administracja centralna <br>Lokacja główna|
|**Sprawdź odwołania do kolekcji niezgodne**|Podczas uaktualniania ten test sprawdza kolekcje odwoływać się tylko inne kolekcje tego samego typu.|Błąd|Centralna lokacja administracyjna|  
|**Wersja klienta na komputerze punktu zarządzania**|Sprawdza, czy punkt zarządzania są instalowany na komputerze, który nie ma innej wersji zainstalowanego klienta programu Configuration Manager.|Błąd|Punkt zarządzania|
|**Konfiguracja użycia pamięci przez program SQL Server**|Sprawdza, czy program SQL Server jest skonfigurowany do użycia pamięci bez ograniczeń. Należy skonfigurować maksymalny limit pamięci programu SQL Server.|Ostrzeżenie|Serwer SQL|
|**Dedykowane wystąpienie programu SQL Server**|Sprawdza, czy dedykowane wystąpienie programu SQL server jest skonfigurowany do obsługi bazy danych lokacji programu Configuration Manager. |**Istniejące składniki serwera programu Configuration Manager na serwerze**|Sprawdza czy serwer lokacji lub rola systemu lokacji nie są już zainstalowane na komputerze wybranym dla instalacji lokacji.|Błąd|Witryny Administracja centralna <br>Lokacja główna <br>Lokacja dodatkowa|
